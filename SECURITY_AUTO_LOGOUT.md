# ملخص نظام الأمان - تسجيل الخروج التلقائي

## 🔐 الموظفين (Dashboard)
- **Heartbeat Interval:** كل 5 ثواني
- **زمن الاكتشاف:** 5 ثواني max
- **الطرد الفوري:** عند أي API request
- **السبب:** الموظفين لديهم صلاحيات حرجة (حذف، تعديل أسعار، إدارة مخزون)

### كيف يعمل:
1. كل 5 ثواني، يُرسل heartbeat تلقائياً
2. إذا تم إلغاء تفعيل الموظف:
   - خلال 5 ثواني max: الـ heartbeat يكتشف الحساب غير نشط
   - فوراً: يتم مسح الـ session و redirect لصفحة Login
3. إذا حاول الموظف أي action (فتح صفحة، حفظ، حذف):
   - **فوراً** (< 1 ثانية): يُطرد من النظام

---

## 👤 المسوقين (Store)
- **Heartbeat Interval:** لا يوجد (ON-DEMAND)
- **زمن الاكتشاف:** عند أول API request
- **الطرد الفوري:** عند أي API request
- **السبب:** المسوقين ليس لديهم صلاحيات حرجة

### كيف يعمل:
1. لا يوجد heartbeat تلقائي
2. عند أول action (فتح صفحة، سحب رصيد، إلخ):
   - السيرفر يفحص إذا الحساب نشط
   - إذا غير نشط: **فوراً** يُطرد من النظام

---

## 🛡️ الحماية الإضافية
- **كل API request** يمر عبر `fetchJson` في `employeeService.ts`
- **403 Forbidden + "غير نشط"** → Logout فوري
- **Session Clear:** كامل (sessionStorage + localStorage)
- **Redirect:** لصفحة Login مع منع الرجوع

---

## 📊 الفرق الرئيسي

| العنصر | الموظفين | المسوقين |
|--------|---------|----------|
| **Heartbeat** | كل 5 ثواني | لا يوجد |
| **أقصى زمن** | 5 ثواني | فوري مع أول action |
| **الصلاحيات** | حرجة جداً | محدودة |
| **الخطورة** | عالية | منخفضة |

---

## ✅ النتيجة النهائية
- **موظف غير نشط** = طرد خلال 5 ثواني max
- **مسوق غير نشط** = طرد فوري مع أول action
- **أمان عالي** = بدون أي ثغرات
